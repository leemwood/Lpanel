# 记忆记录

## 2026-02-04
- 用户要求基于 Golang 制作一个网站启动器（类似 1panel/宝塔）。
- 确定技术方案：Golang + Gin + SQLite + 原生前端。
- Nginx 作为可选内置组件。
- 用户反馈功能太单一，计划扩展以下功能：
  1. 系统状态监控（CPU、内存、磁盘、运行时间）。
  2. 基础文件管理器（浏览服务器目录）。
  3. UI 界面升级（侧边栏导航，仪表盘视图）。
  4. 进程 management 基础。
- 修复了 `files.map` 错误及路径切换页面不刷新问题。
- 实现了创建网站时的目录选择器功能，通过通用的目录选择模态框提升用户体验。
- 引入了内置站点运行引擎 (Site Runner)，支持多端口监听、域名分发及反向代理，修复了 80 端口无法访问的问题。
- 配置了 GitHub Actions CI/CD 流水线，支持自动构建 Windows/Linux 双版本并上传构建产物（Artifacts/Release）。
