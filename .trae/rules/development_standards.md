## 一、系统与环境配置
1. **操作系统**：Windows Server 2022
2. **终端命令分隔符**：使用分号（;）
3. **默认语言**：简体中文
4. **禁用表情符号**：所有输出禁止使用emoji

## 二、开发规范（强制要求）
1. **前端样式分离**
   - HTML、JavaScript 与 CSS 必须分离
   - 所有样式统一写入独立的 `.css` 文件
   - 禁止内联样式和嵌入式样式表
   
2. **存储空间管理**
   - C盘空间有限，所有项目构建依赖、缓存必须设置在E盘
   - 构建工具需配置为使用E盘路径

## 三、工具与路径配置
1. **Maven**
   - 安装路径：`E:\apache-maven-3.9.12-bin\apache-maven-3.9.12\bin`
   - 执行 Maven 命令前必须设置环境变量

2. **MinGW64**
   - 环境路径：`E:\mingw64\bin`
   - 按需调用，非必要不使用

3. **Java 环境**
   - JDK 21 和 25 均安装在 E 盘
   - 根据项目需求选择对应版本

## 四、工作流程规范
1. **方案确认机制**
   - 执行任务前必须评估是否需要先制定详细方案
   - 如需方案，需提供完整方案供用户确认或修改后方可执行
   - 方案应包括：步骤、工具、预期结果、风险点

2. **工具调用说明**
   - 每次调用任何工具都必须简要说明理由
   - 说明内容不记录到规则文件

3. **搜索与信息获取**
   - 首选常规搜索工具
   - 如搜索结果不准确，使用 MCP 浏览器工具访问 Bing 搜索
   - 可获取特定网站的详细内容（谷歌无法访问）

## 五、文档与记录管理
1. **规则文件夹结构**（位于项目根目录的 `.trae/rules/`）
   ```
   .trae/rules/
   ├── development_standards.md    # 开发规范要求
   ├── memory_log.md              # 记忆记录
   └── knowledge/                 # 知识点目录
   ```

2. **知识点文档规范**
   - 每个知识点创建独立 `.md` 文件
   - 文件头部必须包含 yaml 配置

3. **记录要求**
   - **记忆记录**：在 `memory_log.md` 中简要记录每次用户要求
   - **知识点记录**：新获取的知识点创建独立文件到 `knowledge/` 目录
   - **开发规范**：本规则文件保存为 `development_standards.md`
